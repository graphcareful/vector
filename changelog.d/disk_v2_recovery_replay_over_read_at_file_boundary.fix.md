Fixed a `disk_v2` buffer bug where an end-to-end-acknowledged event could be permanently lost after a crash. On restart the reader replays the already-acknowledged records of its resume data file to reposition to the durable read frontier. When that frontier sat exactly at the boundary of a prior data file that had since been deleted, the resume file began with the first *unread* record; the replay read it anyway, counted its bytes as already-consumed, and advanced past it, so it was never delivered even though it was still durably on disk. This most often struck large records, which are written to their own data file and therefore sit alone at such a boundary. The reader now peeks the first record id of each data file before replaying it and stops when the file already begins past the frontier, so the first unread record is delivered instead of being skipped.
