Fixed a `disk_v2` buffer bug where an end-to-end-acknowledged event could be permanently lost after a crash. When the reader finished a data file it unlinked the file before durably recording that it had advanced past it, so a crash in that window left the ledger's reader position pointing at a file that no longer existed; on reopen the reader could skip over the still-durable records in that file (acknowledging them as a gap) and drop them. The reader now advances and flushes its position in the ledger *before* unlinking a completed data file, so a crash in the window leaves a harmless orphaned file rather than a lost position. On reopen the buffer reclaims any such orphaned files (fully acknowledged files behind the durable reader position) so they neither inflate the recomputed buffer size nor leak disk space.
