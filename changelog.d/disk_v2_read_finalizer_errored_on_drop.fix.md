Fixed a `disk_v2` buffer bug where a record read from the buffer could be treated as delivered when it never was. The buffer attaches a finalizer to each record it hands downstream; if that finalizer was dropped without any acknowledgement — for example, when the sink task is torn down with a request still in flight around a crash — it finalized with the default "dropped" status, which the batch treats as a no-op, leaving the batch at its optimistic `Delivered` default. The buffer's read position was then advanced past a record that was never delivered. The read path now uses a pessimistic finalizer, so a dropped, unacknowledged delivery is reported as a failure (`Errored`) instead of silently counting as delivered, mirroring the write path's existing errored-on-drop guard.
